# レスポンシブデザイン最適化チェックリスト

## 実装済み項目

### 1. タッチターゲットのサイズ（44x44px以上）
- ✅ ボタンコンポーネント: `min-h-[44px] min-w-[44px]` を追加
- ✅ アイコンボタン: すべて44x44px以上に調整
- ✅ 検索バーのクリアボタン: 44x44px以上に調整
- ✅ チャット入力の送信ボタン: 44x44px以上（icon size）

### 2. フォントサイズの最適化（最小16px）
- ✅ 入力フィールド: iOS Safariでの自動ズーム防止のため16pxに設定
- ✅ タブレット以上: 14pxに設定（メディアクエリ）
- ⚠️ 補助テキスト（text-xs）: 12pxのまま（装飾的な要素のため許容）

### 3. スクロールの最適化
- ✅ スムーズスクロール: `scroll-behavior: smooth` を追加
- ✅ フォントレンダリング: アンチエイリアス設定

### 4. 画像の最適化
- ⚠️ Avatarコンポーネント: Radix UIのAvatarを使用（Next.js Imageへの置き換えは将来実装）
- ℹ️ 現在、アバター画像は外部URLを使用しており、Next.js Imageの最適化は不要

## 手動確認が必要な項目

### 1. モバイル（375px）での表示確認
以下の画面を375px幅で確認してください：

1. **ホーム画面** (`/`)
   - 検索バーとフィルタボタンの配置
   - 投稿カードのレイアウト
   - 無限スクロールの動作

2. **投稿詳細画面** (`/posts/[id]`)
   - ヘッダーの戻るボタン
   - 応募ボタンのサイズ
   - 応募者リストの表示

3. **投稿作成画面** (`/posts/new`)
   - フォーム入力フィールドのサイズ
   - 送信ボタンの配置

4. **チャット画面** (`/chat`, `/chat/[id]`)
   - チャットルームカードのサイズ
   - メッセージ入力エリア
   - 送信ボタンのサイズ

5. **プロフィール画面** (`/profile`)
   - プロフィールヘッダー
   - タブの切り替え
   - 編集ボタン

### 2. パフォーマンステスト（Lighthouse）
以下のコマンドでLighthouseテストを実行してください：

```bash
# Chrome DevToolsを使用
# 1. Chromeでアプリを開く
# 2. DevToolsを開く（F12）
# 3. Lighthouseタブを開く
# 4. 「モバイル」を選択
# 5. 「パフォーマンス」「アクセシビリティ」「ベストプラクティス」「SEO」を選択
# 6. 「分析」をクリック

# または、CLIを使用
npx lighthouse http://localhost:3000 --view --preset=mobile
```

**目標スコア:**
- パフォーマンス: 90以上
- アクセシビリティ: 90以上
- ベストプラクティス: 90以上
- SEO: 90以上

### 3. 実機テスト
以下のデバイスで実機テストを推奨：

- iPhone SE (375px)
- iPhone 12/13/14 (390px)
- Android スマートフォン (360px - 412px)

**確認項目:**
- タッチターゲットが適切なサイズか
- フォントが読みやすいか
- スクロールがスムーズか
- 入力フィールドで自動ズームが発生しないか

## 改善が必要な項目（将来実装）

### 1. Next.js Imageコンポーネントの使用
現在、AvatarコンポーネントはRadix UIを使用していますが、将来的にはNext.js Imageコンポーネントを使用して画像最適化を行うことを推奨します。

### 2. 画像の遅延読み込み
投稿カードに画像を追加する場合、`loading="lazy"`属性を使用して遅延読み込みを実装します。

### 3. レスポンシブ画像
異なる画面サイズに応じた画像サイズを提供するため、`srcset`属性を使用します。
