# 手動テストチェックリスト

## 1. 認証フローのテスト

### 1.1 ログイン
- [ ] メールアドレスとパスワードでログインできる
- [ ] 間違った認証情報でエラーメッセージが表示される
- [ ] ログイン後、ホーム画面にリダイレクトされる
- [ ] ログイン状態が保持される（ページリロード後も）

### 1.2 新規登録
- [ ] 新規登録フォームが表示される
- [ ] メールアドレスとパスワードで登録できる
- [ ] 登録後、オンボーディング画面にリダイレクトされる
- [ ] 既存のメールアドレスでエラーメッセージが表示される

### 1.3 オンボーディング
- [ ] 初回ログイン時にオンボーディング画面が表示される
- [ ] プロフィール情報（名前、自己紹介、アバターURL）を入力できる
- [ ] オンボーディング完了後、ホーム画面にリダイレクトされる
- [ ] 既にプロフィールが設定されている場合はスキップされる

### 1.4 認証状態の確認
- [ ] 未認証ユーザーは保護されたページにアクセスできない
- [ ] 未認証ユーザーはログイン画面にリダイレクトされる
- [ ] ログアウト機能が動作する（将来実装）

## 2. 投稿CRUDのテスト

### 2.1 投稿一覧表示
- [ ] ホーム画面で投稿一覧が表示される
- [ ] 投稿カードに必要な情報（タイトル、作者、日時、場所、人数）が表示される

- [ ] バリデーションが動作する（文字数制限、日時チェック）
- [ ] 投稿作成後、投稿詳細画面にリダイレクトされる
- [ ] エラーメッセージが適切に表示される

### 2.3 投稿詳細表示
- [ ] 投稿カードをクリックして詳細画面に遷移できる
- [ ] 投稿の全情報が表示される
- [ ] 作者情報が表示される
- [ ] タグが表示される
- [ ] 存在しない投稿IDで404エラーが表示される

### 2.4 投稿編集
- [ ] 自分の投稿のみ編集ボタンが表示される
- [ ] 編集フォームが表示される
- [ ] 既存の値がフォームに反映される
- [ ] 編集内容が保存される
- [ ] 他人の投稿は編集できない

### 2.5 投稿削除
- [ ] 自分の投稿のみ削除ボタンが表示される
- [ ] 削除確認ダイアログが表示される
- [ ] 削除後、投稿一覧から非表示になる（論理削除）
- [ ] 他人の投稿は削除できない

## 3. 応募フローのテスト

### 3.1 応募作成
- [ ] 投稿詳細画面で「応募する」ボタンが表示される
- [ ] 応募フォームが開く
- [ ] 応募メッセージ（任意）を入力できる
- [ ] 応募が正常に送信される
- [ ] 応募後、トースト通知が表示される
- [ ] 自分の投稿には応募できない
- [ ] 既に応募済みの場合は「応募済み」と表示される
- [ ] 募集終了の場合は応募できない
- [ ] 満員の場合は応募できない

### 3.2 応募一覧表示
- [ ] 投稿詳細画面で応募者一覧が表示される（投稿者のみ）
- [ ] 応募者情報（名前、アバター、メッセージ）が表示される
- [ ] 応募ステータス（承認待ち、承認済み、却下）が表示される
- [ ] 応募がない場合、空状態が表示される

### 3.3 応募承認/却下
- [ ] 投稿者のみ承認/却下ボタンが表示される
- [ ] 承認ボタンで応募を承認できる
- [ ] 却下ボタンで応募を却下できる
- [ ] 承認後、トースト通知が表示される
- [ ] 承認後、チャットルームが作成される
- [ ] 承認済みの応募は承認/却下できない

### 3.4 応募キャンセル
- [ ] 応募者が自分の応募をキャンセルできる
- [ ] キャンセル確認ダイアログが表示される
- [ ] キャンセル後、トースト通知が表示される
- [ ] 承認済みの応募はキャンセルできない（投稿者のみキャンセル可能）
